# 0.1.3 — 2026-07-16

## Added

- **`nps_get_alerts` `start`** — zero-based pagination offset mirroring the other list tools, so alerts past the first page are reachable ([#6](https://github.com/cyanheads/national-parks-mcp-server/issues/6)).
- **Socket security scanner** — `@socketsecurity/bun-security-scanner` wired into `bunfig.toml` `[install.security]` to scan every package before install, alongside a `minimumReleaseAge` 3-day install guard (`@cyanheads/mcp-ts-core` excluded so framework releases adopt same-day).
- **`.github/SECURITY.md`, `.github/FUNDING.yml`, `.gitattributes`** — vulnerability-disclosure policy, funding links, and LF normalization plus `linguist-generated` markers for `bun.lock` / `CHANGELOG.md` / `docs/tree.md`.

## Changed

- **`Dockerfile`** — pinned `oven/bun:1.3.14` on the build and production stages, added BuildKit cache mounts to the install steps, and `--ignore-scripts` to the build-stage install.
- **`scripts/` + `skills/`** — re-synced to the `@cyanheads/mcp-ts-core` 0.10.14 baseline; `CLAUDE.md` / `AGENTS.md` command tables moved from `npm` to `bun` and the skill registry gained `techniques` and `api-mirror`.
- **`devcheck.config.json`** — `outdated.allowlist` now holds `typescript` (pinned behind its latest major); `@socketsecurity/bun-security-scanner` added to the depcheck ignore set.
- **`package.json`** — description records the STDIO / Streamable HTTP transports, `author` collapsed to a single string, and `packageManager` pinned to `bun@1.3.14`.

## Fixed

- **Local filters returned false-empty results** — `nps_find_parks.activity` and `nps_get_alerts.category` filtered a single upstream page, so a small `limit` could report zero matches while later pages matched. Both now fetch the whole set matched by the other filters (from offset 0) and slice locally, so `totalCount` is the true post-filter count ([#1](https://github.com/cyanheads/national-parks-mcp-server/issues/1)).
- **Truncation is start-aware** — the "more pages exist" test is now `start + shown < total` (was `total > shown`, which misfired on the final page and advertised a next page that did not exist). Every list tool's truncation guidance names the next `start` — or `pageNumber` for `/events` ([#6](https://github.com/cyanheads/national-parks-mcp-server/issues/6)).
- **`nps_get_alerts` ordering diverged by client surface** — `format()` re-sorted by category while the handler sorted by recency, so `structuredContent` and `content[]` clients received different orders. `format()` now renders the handler's most-recent-first order; the per-row `[category]` heading keeps severity legible ([#7](https://github.com/cyanheads/national-parks-mcp-server/issues/7)).
- **`nps_get_alerts` reported a false all-clear on an empty page** — `format()` asserted "nothing closed or hazardous" for any empty result, but it never sees `totalCount` and could not tell an all-clear from a page past the end. It now defers to the notice, which branches on `totalCount` ([#10](https://github.com/cyanheads/national-parks-mcp-server/issues/10)).
- **Empty page vs. genuine no-match** — an empty page past the end of a non-empty matched set is now distinguished from a real no-match across all five list tools; the empty-result notice branches on `totalCount` and points back to `start=0` (or `pageNumber=1`).
