# 0.1.4 — 2026-07-16

## Added

- **`nps_get_park` `imagesTruncated`** — additive optional boolean on each park, `true` when the upstream image list exceeded the `MAX_IMAGES_PER_PARK` (5) cap; the park `url` is the path to the rest. Present only when the `images` section is included ([#5](https://github.com/cyanheads/national-parks-mcp-server/issues/5)).

## Changed

- **`nps_find_parks` ranks `query` matches by relevance** — NPS `/parks` returns matches alphabetical-by-code with no relevance order, so the obvious park could sit behind description-only matches, even off the first page. Results are now re-ranked over the full matched set before the page is sliced, by transparent tiers — exact `parkCode` → exact name → name prefix → term elsewhere in name → description-only — preserving NPS's upstream order within each tier ([#4](https://github.com/cyanheads/national-parks-mcp-server/issues/4)).

## Fixed

- **`nps_find_parks` activity parity** — `format()` capped the text channel at the first 8 activities ("+ N more") while `structuredContent` carried the full array, so `content[]`-only clients silently saw fewer. Both channels now render the complete list ([#5](https://github.com/cyanheads/national-parks-mcp-server/issues/5)).
- **`nps_get_park` image parity** — `format()` rendered only the first image while `structuredContent` carried up to 5. The text channel now renders every returned image, adding a disclosure line when `imagesTruncated` is set ([#5](https://github.com/cyanheads/national-parks-mcp-server/issues/5)).
