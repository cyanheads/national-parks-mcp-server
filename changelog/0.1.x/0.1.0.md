# 0.1.0 — 2026-06-13

Initial release. An MCP server for planning US National Park Service trips, wrapping the NPS Data API (`developer.nps.gov/api/v1`, `X-Api-Key` auth). US NPS sites only — not state parks or Forest Service / BLM land.

## Added

- **`nps_find_parks`** — resolve a place name, US state, or free-text query to parks; returns the `parkCode` spine plus a trip-planning summary. Local `activity` filter over the returned page.
- **`nps_get_park`** — full detail for up to ten parks in one batched call: description, activities/topics, fees & passes, operating hours, contacts, directions, weather overview, images. `fields` trims the payload; unresolved codes surface as `missingCodes`.
- **`nps_get_alerts`** — current alerts for a park or state with category and recency surfaced first, sorted most-recent-first, Danger/Park Closure ordered ahead. Empty result framed as good news.
- **`nps_find_campgrounds`** — campgrounds with amenities (water, showers, dump station, RV, toilets, trash), reservable vs. first-come site counts, reservation info, accessibility, and fees.
- **`nps_get_activities`** — curated things to do (`/thingstodo`): title, duration, location, accessibility, fee/pet/reservation flags. Single-string `parkCode`/`stateCode`; one location filter required.
- **`nps_find_events`** — scheduled events within a date range with dates/times, location, category, fee, and registration links. Page-based pagination; upstream `errors[]` folds into the result notice.
- **`NPS_API_KEY`** (required) and **`NPS_BASE_URL`** (optional) configuration; server fails to start without a key.

## Notes

The NPS API returns many numeric and boolean fields as strings, nests some values (e.g. `campsites.totalSites`), and uses a distinct envelope and lowercased field names for `/events`. The service layer normalizes all of it — coordinates and site counts to numbers, `"true"`/`"false"`/`"1"`/`"0"` and array amenity fields to booleans, empty strings to null, alert dates to `YYYY-MM-DD`, and event descriptions to HTML-stripped plain text — so tool handlers receive clean domain types.
