# 0.1.2 — 2026-06-20

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.10.6` → `^0.10.9` — adopts the framework's fresh-scaffold devcheck guards (changelog-sync, git-dependent checks, worktree-deletion guard), the `DuckdbProvider.describe()` filter-qualification fix, the `ctx.content` collector for image/audio blocks, Canvas SQL `invalid_sql` prepare-failure classification, and the two new static devcheck guards (floating dependency specifiers, plugin marketplace manifest correctness). No tool or behavior change here — this server exposes none of the touched surfaces.
- **`devcheck.config.json`** — added `packaging.pluginManifests: true`, enabling `lint:packaging` check 10 to validate `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json` (non-empty descriptions, unscoped display identity, full-name install args).
- **`scripts/`** — re-synced six framework devcheck scripts to the 0.10.7–0.10.9 baseline: new `check-dependency-specifiers.ts` (hard-fails on `latest`/`*`/dist-tag specifiers in `package.json` and `bun.lock` workspaces), plus `build-changelog.ts`, `check-framework-antipatterns.ts`, `check-skill-versions.ts`, `devcheck.ts`, `lint-packaging.ts`.
- **Project skills** — re-synced fourteen `skills/<name>/SKILL.md` files to the framework 0.10.7–0.10.9 versions.
- **`@types/node`** `25.9.3` → `26.0.0` (dev).
